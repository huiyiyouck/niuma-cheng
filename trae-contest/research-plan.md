# TRAE AI 创造力大赛参赛调研方案

调研日期：2026-07-02  
参考链接：https://forum.trae.cn/t/topic/22549

## 1. 比赛要求拆解

根据 TRAE 官方社区《初赛参赛指南》，初赛阶段为 2026-06-16 至 2026-07-15，与报名同步进行。有效参赛的前提是报名审核通过。

初赛作品的核心不是复杂度，而是：

- 有可交互、可体验的 Demo。
- 能清楚说明真实用户、真实痛点和核心价值。
- 能证明作品基于 TRAE IDE 或 TRAE Work 完成。
- 作品帖至少包含 Demo 简介、创作思路、体验地址、TRAE 实践过程。
- 需要提供不少于 3 张关键步骤截图，以及不少于 3 个关键任务 Session ID。
- 可提交多个 Demo；但每个初赛 Demo 都应与报名创意方向一致，帖子标签也需要与报名通过的赛道一致。
- 初赛 Demo 后续复赛可以演进成更完整产品。
- 初赛结果公示时间为 2026-07-21 至 2026-07-23，提交后应预留复赛准备节奏。

这意味着本地已有项目可以参赛，但必须把“创意、体验、过程证据、参赛期内实质性迭代”整理清楚。

## 2. 本地资产盘点

`/Users/ck/Project/niuma-cheng` 是一个多仓生态根目录，包含：

- `niuma-cheng-xiaobao`：AI 多源新闻聚合平台，已部署 `news.huiyiyou.cloud`。
- `niuma-cheng-ai`：AI 处理中枢 / Agent Hub，已实现 news-l1 真实处理链路，测试环境联调通过。
- `niuma-cheng-workboard`：跨项目 Agent 工作看板，v0.1 已上线 `workboard.huiyiyou.cloud`。
- `niuma-cheng-coordination`：跨项目契约、状态、需求、决策真源。
- `agent-workflow`：一人公司 AI 开发团队工作流真源。

这些资产天然分成两类：

1. 面向普通用户/信息工作者的产品：小报 + AI。
2. 面向 AI 开发者/创作者的方法论产品：工作流 + coordination + workboard。

## 3. 候选项目可行性判断

### 方案 A：小报 + AI 作为主参赛项目

结论：高度可行，建议作为第一优先级。

优势：

- 有明确用户价值：把多源信息自动抓取、分类、展示、评分，并逐步接入 L1 AI 处理。
- 有线上体验入口：`news.huiyiyou.cloud`，符合“可体验 Demo”的硬要求。
- 有足够过程证据：v0.4 到 v0.6 有 PRD、设计、实现、部署、测试、问题修复和跨项目联调记录。
- 有参赛期内实质性迭代：2026-06-16 到 2026-07-01 期间，独立 AI Hub、跨项目契约、`/debug/ai` 联调页、KB search、生产隔离部署等都可构成“用 TRAE 继续推进作品”的证据。

风险：

- 如果只讲“新闻聚合”，会显得赛道常见；必须突出“个人信息处理流水线 + AI Agent 化处理 + 可运营的信息源系统”。
- AI 处理在生产默认关闭，参赛帖要诚实说明：线上主站可体验新闻产品，AI 联调能力可通过测试环境/截图/演示视频展示。
- 需要把敏感配置、内部 token、数据库细节从截图里处理掉。

推荐定位：

“面向个人创业者和 AI 从业者的智能信息中枢：自动抓取多源信号，进行分类、去重、评分和 AI 摘要，帮助一个人持续跟踪 AI/财经/Web3 情报。”

### 方案 B：AI 工作流 / 一人公司开发团队作为第二参赛项目

结论：可行，但需要包装成“可体验工具”，不能只提交方法论文档。

优势：

- 创意差异化强：不是单个 App，而是一套把 PM、Architect、Developer、DevOps、Review、跨项目协作组织起来的 AI 工作系统。
- 证据丰富：`agent-workflow`、`coordination`、`workboard` 已形成闭环，并在小报、AI Hub、工作看板项目中真实使用。
- 已有可视化入口：`workboard.huiyiyou.cloud`，可作为 Demo 体验面。

风险：

- 评审可能不愿意读复杂流程文档；必须用 workboard 把“工作流如何让项目稳定推进”可视化。
- 如果把它讲成“我有一套流程”，会偏知识分享；要讲成“一个可复制的 AI 项目操作系统”。
- 需要补一段非常短的演示路径：创建/接入项目、查看角色状态、发现阻塞、追踪跨项目需求。

推荐定位：

“一人公司 AI 项目操作系统：用多角色 AI 工作流、跨项目契约和只读工作看板，把一个人的多个 AI 项目组织成可追踪、可复盘、可交接的开发生态。”

### 方案 C：只报 AI Hub

结论：不建议单独作为初赛主项目。

原因：

- 它更像小报背后的能力模块，单独展示用户价值不如小报直观。
- 目前主要价值在跨项目服务化和真实 L1 处理链路，评审成本偏高。
- 更适合作为小报项目的“AI 能力升级”部分。

## 4. 报名前提核实

在动手包装截图、Session ID 和 Demo 帖之前，必须先核实报名记录。这是专业评审通道的合规门禁。

需要确认：

- 已通过报名的创意方向是什么。
- 报名通过的赛道标签是什么。
- 是否有一条报名，还是两条独立报名。
- 如果只有一条报名，报名方向是否足以覆盖“小报 + AI”和“工作流 + Workboard”两个 Demo。

判断规则：

- 如果已有两条独立报名，且分别对应“小报 + AI”和“工作流 + Workboard”，可以按双作品推进。
- 如果只有一条报名，优先提交与报名方向最一致的作品；另一个作品只能作为背景材料、创作过程或后续复赛扩展，不能默认作为独立 Demo 参评。
- 如果报名赛道不是“学习工作”，下面所有标题和标签必须改成实际报名赛道。

## 5. 推荐参赛组合

在报名方向确认匹配的前提下，建议提交两个 Demo：

1. 主作品：牛马程小报 + AI 信息处理中枢
   - 赛道：以报名通过的赛道为准；若报名为学习工作，可使用学习工作。
   - 体验入口：`news.huiyiyou.cloud`，补充测试环境 `/debug/ai` 或演示视频。
   - 参赛主张：一个人也能拥有持续运行的信息情报台。

2. 第二作品：一人公司 AI 工作流 + 跨项目 Workboard
   - 赛道：以报名通过的赛道为准；若有独立报名且报名为学习工作，可使用学习工作。
   - 体验入口：`workboard.huiyiyou.cloud`。
   - 参赛主张：把 AI 编程从“聊天写代码”升级为“可治理的个人研发系统”。

如果报名条件允许两个 Demo 独立参评，它们可以互相加分：小报证明工作流能产出真实产品；工作流解释小报为什么能稳定迭代。

如果报名条件只允许一个 Demo，建议优先报“小报 + AI”，并把工作流作为“TRAE 实践过程”和“项目管理方法”写进创作过程，不单独发第二个初赛作品帖。

## 6. 7 月 15 日前行动清单

- 找到已通过的报名帖链接，确认报名创意方向和赛道。

- 确认主站公开访问正常，首页、管理页、新闻详情、信息源管理路径可演示。
- 准备一条 3 分钟演示路径：创建/查看信息源 → 浏览新闻 → 查看评分/标签 → 展示 AI debug 或 AI 处理结果。
- 补一组参赛专用截图：产品界面 3 张、TRAE 关键任务截图 3 张以上、跨项目联调证据 1-2 张。

### 工作流 + Workboard

- 确认 `workboard.huiyiyou.cloud` 可公开访问，关键页面无内部路径泄露。
- 准备一条 2 分钟演示路径：项目总览 → 角色状态 → 跨项目需求 → 阻塞/下一步。
- 把 `agent-workflow` 的价值压缩成 3 句话，不让评审读长文档。
- 补一张“生态关系图”：小报、AI Hub、coordination、workboard、agent-workflow 的关系。
- 整理 3 个 TRAE Session ID：工作流基线设计、workboard 实现、跨项目协调机制。

### 可选：抖音人气通道

- 如果精力允许，为最终提交的 Demo 准备一条 30-60 秒短视频或图文。
- 发布时带话题 `#vibecoding大赏`、`#traeai创造力大赛`，并按官方要求 `@TRAE`、`@抖音科技`。
- 抖音通道独立于专业评审，但仍要求先在社区提交合规 Demo 帖；不要把它当作替代报名或替代社区作品帖。

### 精力预算

当前距离初赛提交截止 2026-07-15 约 13 天，建议按报名情况分配：

- 单作品策略：70% 时间打磨“小报 + AI”的体验、截图、Session ID、参赛帖；20% 时间整理工作流作为过程证据；10% 时间准备抖音可选材料。
- 双作品策略：60% 时间给“小报 + AI”；30% 时间给“工作流 + Workboard”；10% 时间做统一脱敏、交叉引用和抖音可选材料。
- 若任一作品无法在 2 天内补齐演示路径、截图和 Session ID，应降级为辅助材料，避免两个作品都变成半成品。

## 8. 最终判断

可行，但“双作品组合”不是无条件成立，前提是报名记录支持两个创意方向分别参评，或同一报名方向能够合理覆盖两个 Demo。

最稳打法是：

- 先核实报名创意方向和赛道。
- 如果只能提交一个强作品，用“小报 + AI”打产品价值和可体验 Demo，把工作流作为 TRAE 实践深度写进过程。
- 如果可以提交两个合规作品，再用“工作流 + Workboard”打创造力、方法论和 TRAE 实践深度。
- 两个帖子之间可以互相引用：小报是工作流产出的真实案例，工作流是小报持续迭代的底层系统。

当前最需要补的不是代码，而是参赛包装：演示路径、截图、Session ID、脱敏、标题和短叙事。
