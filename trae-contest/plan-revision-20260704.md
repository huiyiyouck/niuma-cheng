# 方案修订单(2026-07-04,指挥官已拍板,待参谋长落入 workflow-demo-plan.md)

> 来源:指挥官与 review 方(Claude 大管家)对齐结论。触发原因:① 指挥官实际使用 TRAE Work 而非 TRAE IDE,自定义智能体+分享链接为 IDE 独有功能(已核官方文档);② 大赛硬要求核实为"开发证据"(Session ID/截图为开发者的,评委无需使用 TRAE),"可体验 Demo"线上入口即可满足。

## 一、核心变更:线 1 主推换血 + 两步走

**原线 1(TRAE 智能体分享链接)降级为可选加分项;新主推:workboard「AI 工作流入口」迭代,两步交付。**

### 第一步:比赛合规交付(建议 07-10 前)

workboard v0.2「AI 工作流入口」MVP 迭代,**在 TRAE Work 里按标准迭代流程开发**(PRD→Review→开发→部署),一次产出:

1. 大赛证据:Session ID(≥3,开发过程自然产生)+ 关键步骤截图(≥3)
2. "工作流产出现实产品"活证据(迭代对象就是参赛作品自身,替代原 X.1 的角色)
3. workboard 真实产品升级(v0.2"管理中枢"方向提前落地,走元信息台账登记)

**MVP 范围(钉死,超出即砍)**:
- 单预置演示项目(虚拟"TODO 应用",内存态,每会话重置),评委选**角色**不选项目
- 单场景:1.7 体验脚本("你是 PM,给 TODO 功能写 PRD 并走 Review")
- 静态门禁回放(照 1.7 脚本做可点击对话回放)——**先做,一天内可交付,公开上线**
- 真 LLM 对话功能:开发完成但**默认关闸**(环境变量开关);关闸时前端只展示静态回放;录屏时本机开闸演示
- PRD 产出渲染在页面侧栏,不接真文件系统;页面明示"演示环境,完整工作流见 Skill 包/GitHub"

**技术形态(已核实)**:workboard 后端为纯 Node http 服务(src/server ~900 行,无框架),线上即此服务;新增 `/api/demo-chat` 路由(SSE 流式)即可,"加路由"量级,预估 3-4 天。

### 第二步:评委试用开闸(纯加分项,时间盒 1-2 天,07-13 前做不完即砍)

- 试用按钮 → 发临时 token(随机 ID + 服务端内存 Map,TTL 30 分钟)
- 四层护栏(全部服务端执行,缺一不开闸):
  1. 发放限流:同 IP 每小时 N 个 token + 全局每日发放上限(参数待指挥官定)
  2. token 会话校验:TTL + 12 轮上限 + 项目/角色白名单
  3. 每日总额度熔断(金额待指挥官定)
  4. 费用告警
- 砍掉时的帖子口径:可体验 = 看板 + 静态回放 + 录屏,硬要求仍全满足;想真试的评委走线 2 Skill 包
